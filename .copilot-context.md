# LazyTracker — Project Context

## Goal
.NET 8 background app using webcam face detection to open McDonald's Romania careers page when the user looks away or at their phone, enforcing study focus.

## Current State
- Two projects: `LazyTracker.Core` (net8.0 class library) + `LazyTracker.App` (net8.0-windows WinForms tray host)
- OpenCvSharp4 + Haar cascade (`haarcascade_frontalface_default.xml`) for face detection
- Two detection modes: **FaceGone** (turned away, 5s) and **LookingDown** (phone, face position drops, 5s)
- Face position tracking with baseline + EMA smoothing + 5s grace period after resume
- Auto-pause after 15s away; resumes with preserved baseline + notification on return
- System tray icon (pause/resume/exit); browser launches with Win32 focus P/Invoke
- Single-file self-contained publish (~102 MB)
- Git repo: `https://github.com/robyerts/lazyStudentEyeTracker.git`

## Decisions + Rationale
- Haar cascade over ML/DNN: lightweight, no model download, ~98% for face presence
- Face position tracking over eye cascade: eye Haar was ~50-70% accurate, constant false positives
- `FaceDropThreshold: 0.02`: user-tuned for eye-level laptop camera with close sitting distance
- Baseline preserved across auto-pause: avoids re-learning and false triggers on return
- DirectShow backend on Windows: more reliable USB camera detection
- EMA smoothing (factor 0.15) on face Y position: filters Haar cascade jitter at low threshold

## Constraints/Assumptions
- Camera index 0, eye-level laptop webcam, user sits close
- Windows-primary (WinForms tray, P/Invoke) but core library is cross-platform
- Cooldown 15s between triggers; auto-pause at 15s away
- `haarcascade_eye_tree_eyeglasses.xml` downloaded but unused — can be removed
